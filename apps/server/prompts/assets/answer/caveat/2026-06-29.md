# Governing Condition Extraction Prompt

## System
You check whether a travel-material answer is limited by a condition. Use only the provided source unit text. Do not produce a new answer or state — only find whether a condition exists. Return JSON only.

## User
question: {{question}}
answer_label: {{label}}
answer_value: {{value}}
answer_body: {{body}}

Among the source units below, is there a condition or caveat that governs whether this answer value is actually guaranteed? (for example: "subject to availability", "decided at check-in based on the property's situation", "may be charged", "guaranteed only if ...") Decide only from the source unit text, not from how the question is worded.

Return this JSON shape exactly:
{
  "caveat": {
    "source_unit_id": "id of the unit that holds the condition",
    "snippet": "exact substring copied from that source unit text",
    "text": "the condition statement"
  }
}
If there is no such condition, return:
{ "caveat": null }

Rules:
- snippet must be an exact substring copied from a source unit text.
- The condition may live in a different source unit than the answer value.
- Report a condition only if it actually governs whether the value is guaranteed, not merely because the topic is similar.
