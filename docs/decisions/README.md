# 결정 기록

이 폴더는 이후 구현 방향에 영향을 주는 선택을 남길 때 쓴다.

decision note는 나중에 다시 쓰이거나 도전받을 선택에만 유용하다. 작은 로컬 판단은 가까운 commit, PR 본문, `docs/work-log.md` entry를 우선한다.

기본은 단일 markdown 파일이다. 다만 배경 재료를 결론과 분리해서 보존해야 하는 결정은 폴더형으로 남긴다.

- `YYYY-MM-DD-결정-제목.md`: 기본 decision note.
- `YYYY-MM-DD-결정-제목/index.md`: 폴더형 decision의 결론과 현재 판단 요약.
- `YYYY-MM-DD-결정-제목/raw.md`: 대화·조사·검토에서 나온 배경 재료. 당시 상황의 생생함을 보존하기 위한 자료다.

## 폴더형 decision

폴더형 decision은 결론과 배경을 분리하기 위한 구조다.

- `index.md`는 결론, 현재 판단, 기각/보류, 검증, 이번 결정 밖을 담는다. 다음 세션은 먼저 `index.md`를 읽는다.
- `raw.md`는 왜 그런 결론이 나왔는지 이해하기 위한 배경 재료와 당시 상황의 생생함을 담는다. 현재 실행 기준이나 작업 대기열이 아니다.
- `raw.md`에 남은 후보, 권고, 표현을 현재 작업에 쓰려면 `index.md`와 현재 코드 상태를 기준으로 다시 판단한다.
- `raw.md`만 읽어야 알 수 있는 결론은 남기지 않는다.

`raw.md`를 쓸 때는 앵커가 될 표현을 줄인다.

- 개인 로컬 폴더명, 내부 작업명, 대화방 이름, 원문 경로는 그대로 쓰지 않는다.
- 필요하면 `최종 합의본`, `원본 대화`, `외부 조사`, `이전 정리`처럼 역할 이름으로 바꾼다.
- 구어적 대화 표현, 개인 정정 표현, 내부 AI/도구 운영 흔적은 공개 문서에서 독립적으로 읽히는 중립 표현으로 바꾼다.
- 폐기된 후보는 현재 후보가 아니라는 점만 짧게 남긴다.

공개 repo에 남길 wording이 애매하면 로컬 `public-doc-wording-reviewer`를 report-only로 사용한다. 이 agent는 문서 수정의 필수 gate가 아니라, 구어 표현과 private source 누수를 점검하는 보조 도구다.

권장 형식:

```md
# YYYY-MM-DD - 결정 제목

## 맥락

## 결정

## 기각 또는 보류

## 검증

## 이번 결정 밖
```

폴더형 `raw.md` 권장 형식:

```md
# Raw Notes - 결정 제목

이 파일은 `index.md` 결정의 배경 재료이며, 당시 상황의 생생함을 보존하기 위한 자료다. 현재 실행 기준이나 작업 대기열이 아니다.

## 왜 raw가 필요한가

## 대화·조사에서 드러난 문제 감각

## 출처와 확인된 사실

## 기각·보류된 후보

## 남은 불확실성

## 재진입 메모
```
