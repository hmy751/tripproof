# 결정 기록

이 폴더는 이후 구현 방향에 영향을 주는 선택을 남길 때 쓴다.

decision note는 나중에 다시 쓰이거나 도전받을 선택에만 유용하다. 작은 로컬 판단은 가까운 commit, PR 본문, `docs/work-log.md` entry를 우선한다.

기본은 단일 markdown 파일이다. 다만 한 결정이 여러 대화, 외부 조사, 보조 에이전트 검토를 거쳐 다음 세션이 배경을 다시 잡아야 할 만큼 커지면 폴더형으로 남길 수 있다.

- `YYYY-MM-DD-결정-제목.md`: 기본 decision note.
- `YYYY-MM-DD-결정-제목/index.md`: 폴더형 decision의 결론과 현재 판단.
- `YYYY-MM-DD-결정-제목/raw.md`: 대화·조사·검토에서 나온 배경 재료. 현재 실행 기준이 아니다.

폴더형 decision을 만들더라도 `raw.md`를 근거 권위나 작업 대기열로 승격하지 않는다. 다음 세션은 `index.md`를 먼저 읽고, 왜 그런 판단이 나왔는지 필요할 때만 `raw.md`를 본다.

권장 형식:

```md
# YYYY-MM-DD - 결정 제목

## 맥락

## 결정

## 기각 또는 보류

## 검증

## 이번 결정 밖
```
