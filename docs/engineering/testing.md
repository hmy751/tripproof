# 테스트와 검증

## 검증의 종류

형식·타입·단위 테스트와 제품 동작 확인은 다른 것이다. 형식이나 단위 테스트 통과를 제품 동작 통과로 말하지 않는다. 자료 QA·LLM·검색을 다룰 땐 실제 자료를 넣고 질문을 돌린 결과를 따로 확인한다.

## LLM·근거 제품의 특수성

이 제품에서 가장 자주 펴 보는 검증 판단이다.

- 같은 입력에 출력이 흔들릴 수 있다. 통제 가능한 곳은 통제하고(낮은 temperature·형식 강제), 통제 불가한 곳은 근거 검증과 관찰로 흡수한다.
- 자료가 바뀌면 답이 바뀌는지 확인한다.
- "근거 있음"이면 그 근거가 원문의 실제 부분인지 확인한다.
- 입력 자료가 더럽다는 것을 전제로 본다 — 추출 실패·깨진 공백·암호화 PDF.
- 날짜·시간·금액 같은 값은 한국어 단위 문자(년·월·일·시·원)까지 고려해 원문과 맞춘다.
- 테스트가 특정 우회(이 하드코딩 값, 이 우회 문자열)를 콕 집어 막는 식으로 늘어나면 증상을 쫓는 중이다. 그 우회를 낳는 결정 시점(프롬프트·계약·파싱)을 고친다.

## 테스트 크기

- 작은 단위 테스트로 흐름을 빠르게 확인한다. 단 test double은 설정으로 고를 수 없게 격리해 제품 기본 경로로 새지 않게 하고, test double 통과를 제품 통과로 보지 않는다.
- 제품 동작은 product를 실제로 구동해 확인한다.
