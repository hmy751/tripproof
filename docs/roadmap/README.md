# docs/roadmap

TripProof 로드맵 버전을 둔다. 로드맵은 고정 계획이 아니라 갈아엎을 수 있는 초안이라, 버전을 파일로 나눠 쌓는다.

- **현재 기준: `v1.md`** (1차 로드맵 · MVP 계획).
- 새 버전을 쓰면 `v2.md`로 추가하고 위 "현재 기준" 줄을 갱신한다. 이전 버전은 지우지 말고 스냅샷으로 남긴다.
- 어휘·상태·흐름·요구·slice 같은 세부 기준은 여기서 재서술하지 않고 `../product-model.md`·`../prd.md`·`../specs/`를 참조한다.

## 참고 자료

아래 문서는 로드맵 버전이 아니다. 특정 기능 후보를 실제로 열 때 현재 코드·spec과 빠르게 맞추기 위한 개발 지도이며, "현재 기준" 줄을 대체하지 않는다.

- `material-conflict-workflow-reference.md` — 겹치는 여행 자료가 같은 항목을 다르게 말할 때, 자료 충돌 workflow를 어떤 제품 흐름과 구현면으로 열지 보는 개발 지도. 현재 작업 지시가 아니라 후보 판단용 참고 문서다.
