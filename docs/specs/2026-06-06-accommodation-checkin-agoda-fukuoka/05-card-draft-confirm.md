# 숙소 체크인 05 - 카드 초안과 직접 확인

상태: client 카드 초안과 직접 확인 흐름 구현.

부모: [숙소 체크인 확인 - Agoda 후쿠오카 예약 PDF](index.md)

## 왜 지금

TripProof는 답변 로그만 쌓는 도구가 아니다. 사용자가 남길 정보를 카드 초안으로 올리고, AI가 PDF에서 못 찾은 값도 직접 확인으로 채울 수 있어야 한다. 이번 단계는 채팅 답변에서 카드 초안으로 넘어가는 흐름을 닫는다.

## 사용자 장면

사용자는 체크인 답변을 보고 `예약 확정서 제시` 안내는 카드로 남기고 싶다. 체크인 시작 시각이 `근거 부족`으로 남은 경우에는 다른 자료나 숙소 앱에서 확인한 뒤 직접 적어두고 싶다.

## Goal

- 체크인 시 예약 확정서 제시 답변을 카드 초안으로 올릴 수 있다.
- 카드 초안에는 일정, 이름, 값, 상태, 출처가 보인다.
- 사용자가 초안의 이름이나 값을 고칠 수 있다.
- 체크인 시작 시각처럼 `근거 부족`인 항목도 직접 값을 입력할 수 있다.
- 직접 입력하거나 수정한 카드는 `직접 확인`으로 구분된다.
- 카드 초안은 대시보드에 자동 확정되지 않는다.

## Rules

- `DraftListPanel`은 빈 화면이 아니라 초안 목록과 편집 상태를 가진다.
- draft state는 채팅 답변의 항목에서 만들어진다.
- `근거 있음` 초안과 `직접 확인` 초안은 출처 표시가 다르다.
- 직접 입력값은 PDF 근거 snippet처럼 보이지 않게 한다.

## Non-goals

- Agoda나 숙소에 메시지를 보내는 기능.
- 모든 카드 편집 UX.
- 대시보드 배치는 06에서 다룬다.
- companion source에서 자동으로 직접 확인 값을 채우기.

## 현재 코드에서 볼 곳

- `apps/client/App.tsx`: 채팅 답변 항목에서 `CardDraft`를 만들고, 초안 state를 관리한다.
- `apps/client/drafts.ts`: 답변 항목의 근거 상태에 따라 `근거 있음` 초안과 `직접 확인` 초안을 만든다.
- `apps/client/components/ChatWorkspace.tsx`: 답변 항목에서 `초안으로 올리기` 또는 `직접 확인으로 남기기` 액션을 제공한다.
- `apps/client/components/DraftListPanel.tsx`: 초안 목록, 일정/이름/값 편집, 출처 pill, 근거 snippet 또는 사용자 입력 표시를 렌더링한다.
- `apps/client/types.ts`: `CardDraft`, `CardDraftSourceKind`, `ChatAnswerItem` client 타입.

## 구현 결과

- `supported`이고 evidence가 있는 답변 항목은 값과 근거를 보존한 `근거 있음` 초안이 된다.
- `missing`, `needs_review`, `conflict` 항목은 사용자가 먼저 `직접 확인으로 남기기`를 눌러 빈 값 초안으로 만든다.
- 초안의 일정, 이름, 값을 수정하면 출처가 `직접 확인`으로 바뀌고 evidence snippet은 제거된다.
- 직접 입력 초안은 PDF 근거 snippet 대신 `사용자 입력`으로 표시한다.
- 초안은 확인 탭 아래 draft state에만 머물며, 대시보드/현장 카드 확정은 06에서 다룬다.

## 이번 AC

1. 체크인 제시물 답변에서 카드 초안을 만들 수 있다.
2. 초안에서 값과 이름을 확인하거나 수정할 수 있다.
3. `근거 부족`인 체크인 시작 시각도 직접 값을 채울 수 있다.
4. 직접 채운 카드는 `직접 확인`으로 표시된다.
5. 초안은 사용자가 확정하기 전까지 대시보드에 보이지 않는다.

## 닫힌 판단

- `근거 부족` 답변은 자동으로 빈 초안을 만들지 않고, 사용자가 `직접 확인으로 남기기`를 눌렀을 때 초안이 된다.
- 카드 출처는 현재 backend response schema에 넣지 않고 client draft state의 `sourceKind`로 둔다. backend의 `EvidenceState`는 답변 근거 상태만 표현한다.
