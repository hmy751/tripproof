# 숙소 체크인 05 - 대시보드와 현장 카드

상태: sub-spec draft.

부모: [숙소 체크인 확인 - Agoda 후쿠오카 예약 PDF](index.md)

## 왜 지금

숙소 체크인 기능의 도착점은 채팅 답변이 아니라 여행 중 다시 볼 수 있는 카드다. 이번 단계는 사용자가 확정한 카드만 대시보드에 올라가고, 현장 저장한 카드만 현장 탭에 보이는 흐름을 닫는다.

## 사용자 장면

사용자는 `체크인 시 예약 확정서 제시` 카드를 대시보드에 올리고, 현장에서 바로 볼 수 있게 저장한다. 체크인 시작 시각은 PDF 밖에서 직접 확인한 뒤 카드로 올릴 수 있다.

## Goal

- 카드 초안을 확정하면 대시보드에 보인다.
- 대시보드는 AI가 자동으로 채우지 않는다.
- PDF 근거 카드와 직접 확인 카드가 구분된다.
- 예약번호, 고객명, 결제 카드, 숙소 연락처, 정확한 주소는 대시보드 자동 카드가 되지 않는다.
- 대시보드 카드에서 `현장 저장`을 누를 수 있다.
- 현장 탭에는 저장한 카드만 보인다.

## Rules

- `board` 탭은 확정 카드 목록을 보여준다.
- `field` 탭은 현장 저장한 카드 목록만 보여준다.
- 답변, 초안, 확정 카드, 현장 저장 카드는 상태가 분리된다.
- 현장 저장은 대시보드 카드의 명시적인 사용자 행동으로 일어난다.
- `근거 있음`, `직접 확인`, `확인 필요` 출처는 카드에서 구분된다.

## Non-goals

- 전체 여행 일정 생성.
- 여러 카테고리 전체 UX.
- 서버 저장/동기화.
- 오프라인 패키지.
- 민감정보 원문 보기 전용 화면.

## 현재 코드에서 볼 곳

- `client/App.tsx`: board/field 탭은 현재 empty view다.
- `client/components/StaticEmptyView.tsx`: 실제 카드 리스트로 바꿀 대상이다.
- `client/components/ViewTabs.tsx`: 확인, 대시보드, 현장 탭 전환은 이미 있다.

## 이번 AC

1. 확정한 카드만 대시보드에 보인다.
2. 확정하지 않은 답변이나 후보는 대시보드에 보이지 않는다.
3. `현장 저장`한 카드만 현장 탭에 보인다.
4. PDF 근거 카드와 직접 확인 카드가 구분된다.
5. 예약번호, 고객명, 결제 카드 같은 민감 필드는 자동 대시보드 카드가 되지 않는다.

## 남은 판단

- 대시보드에서 일정과 카테고리를 얼마나 단순하게 시작할지.
- 현장 저장 버튼을 대시보드 카드 안에 둘지, 카드 상세 안에 둘지.
- 민감정보를 "원문에서 직접 확인" 링크로만 남길지.
