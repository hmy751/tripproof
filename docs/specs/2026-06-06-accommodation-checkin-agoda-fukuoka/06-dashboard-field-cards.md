# 숙소 체크인 06 - 대시보드와 현장 카드

상태: client 대시보드와 현장 카드 흐름 구현.

부모: [숙소 체크인 확인 - Agoda 후쿠오카 예약 PDF](index.md)

## 왜 지금

숙소 체크인 기능의 도착점은 채팅 답변이 아니라 여행 중 다시 볼 수 있는 카드다. 이번 단계는 사용자가 확정한 카드만 대시보드에 올라가고, 현장 저장한 카드만 현장 탭에 보이는 흐름을 닫는다.

## 사용자 장면

사용자는 `체크인 시 예약 확정서 제시` 카드를 대시보드에 올리고, 현장에서 바로 볼 수 있게 저장한다. 체크인 시작 시각이 `근거 부족`으로 남은 경우에는 직접 확인한 뒤 카드로 올릴 수 있다.

## Goal

- 카드 초안을 확정하면 대시보드에 보인다.
- 대시보드는 AI가 자동으로 채우지 않는다.
- PDF 근거 카드와 직접 확인 카드가 구분된다.
- 대시보드 카드에서 `현장 저장`을 누를 수 있다.
- 현장 탭에는 저장한 카드만 보인다.

## Rules

- `board` 탭은 확정 카드 목록을 보여준다.
- `field` 탭은 현장 저장한 카드 목록만 보여준다.
- 답변, 초안, 확정 카드, 현장 저장 카드는 상태가 분리된다.
- 현장 저장은 대시보드 카드의 명시적인 사용자 행동으로 일어난다.
- `근거 있음` 카드와 `직접 확인` 카드는 구분된다.
- `확인 필요`는 답변 레벨의 근거 상태이며, 카드 출처로 새로 쓰지 않는다.

## Non-goals

- 전체 여행 일정 생성.
- 여러 카테고리 전체 UX.
- 서버 저장/동기화.
- 오프라인 패키지.

## 현재 코드에서 볼 곳

- `apps/client/App.tsx`: `CardDraft` 확정, `DashboardCard` state, 현장 저장 state를 관리한다.
- `apps/client/cards.ts`: 초안을 대시보드 카드로 복사하고 현장 저장 overlay를 만든다.
- `apps/client/components/DraftListPanel.tsx`: 초안에서 `대시보드에 올리기` 액션을 제공한다.
- `apps/client/components/CardCollectionPanel.tsx`: 대시보드와 현장 카드 목록을 렌더링한다.
- `apps/client/components/ViewTabs.tsx`: 확인, 대시보드, 현장 탭 전환은 이미 있다.

## 구현 결과

- 초안은 `대시보드에 올리기`를 눌렀을 때만 `DashboardCard`로 복사된다.
- 확정된 초안은 draft 목록에서 빠지고, 확정 전 답변/초안은 대시보드에 표시되지 않는다.
- 대시보드 카드는 `근거 있음` 카드와 `직접 확인` 카드를 출처 pill과 source 영역으로 구분한다.
- 사용자가 초안의 일정, 이름, 값을 수정해 `manual`로 바꾼 뒤 확정하면 대시보드에서도 `직접 확인` 카드로 유지된다.
- 대시보드 카드의 `현장 저장`을 눌러야 `fieldSavedAt`이 생기고, 현장 탭은 이 카드만 필터링해 보여준다.
- 서버 저장/동기화는 제외하고 client state 기준 MVP 범위로 구현했다.

## 이번 AC

1. 확정한 카드만 대시보드에 보인다. — 구현됨.
2. 확정하지 않은 답변이나 후보는 대시보드에 보이지 않는다. — 구현됨.
3. `현장 저장`한 카드만 현장 탭에 보인다. — 구현됨.
4. PDF 근거 카드와 직접 확인 카드가 구분된다. — 구현됨.

## 실행 확인

- `npm run client:typecheck`
- `npm run build`
- `npm test`

현재 repo에는 client 클릭 흐름을 자동화하는 e2e 테스트가 없으므로, 화면 동작은 다음 수동 흐름으로 확인한다.

1. 확인 탭에서 답변 항목을 초안으로 올린다.
2. 대시보드 탭이 아직 비어 있는지 본다.
3. 초안에서 값을 확인하거나 직접 입력한 뒤 `대시보드에 올리기`를 누른다.
4. 대시보드에서 `근거 있음` 또는 `직접 확인` 카드가 구분되어 보이는지 본다.
5. 대시보드 카드에서 `현장 저장`을 누른다.
6. 현장 탭에 저장한 카드만 보이는지 본다.

## 닫힌 판단

- 대시보드 category는 이번 MVP에서 `숙소` 고정 라벨로 시작한다. 카테고리 편집 UX는 여러 카테고리 확장 때 다시 판단한다.
- 현장 저장 버튼은 카드 상세를 만들지 않고 대시보드 카드 안에 둔다.

## 남은 판단

- 대시보드 카드의 서버 저장/동기화 계약.
- 여러 일정과 카테고리를 사용자가 편집하는 UX.
- 현장 카드의 오프라인 패키지나 정렬 규칙.
