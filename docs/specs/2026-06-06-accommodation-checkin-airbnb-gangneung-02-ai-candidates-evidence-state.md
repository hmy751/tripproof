# 숙소 체크인 02 - AI 해석과 근거 상태

상태: sub-spec draft.

부모: [숙소 체크인 확인 - Airbnb 강릉 숙소 파일](2026-06-06-accommodation-checkin-airbnb-gangneung.md)

## 왜 지금

파일을 파싱해도 해석 결과가 고정값이면 제품 흐름이 아니다. 이번 단계는 01에서 파싱한 본문을 읽어 체크인 후보와 근거 상태가 만들어지는 데까지를 닫는다.

## 사용자 장면

사용자는 자료함에 넣은 숙소 파일을 바탕으로 체크인 시작 시간과 늦은 도착 조건을 확인하고 싶다.

## 이번에 닫을 것

- AI 해석 입력은 01에서 파싱된 파일 본문을 받는다.
- 체크인 시작 시간 `15:00` 후보가 나온다.
- 체크인 후보에는 실제 본문 일부가 근거로 붙는다.
- 늦은 도착 조건은 현재 자료에 값이 없으므로 `근거 부족`으로 남는다.
- 자료에 없는 `22:00 이후 연락`, `셀프 체크인 가능` 같은 값을 만들지 않는다.
- 화면이나 답변은 후보와 상태를 받아서 보여준다.

## 기술 기준

- 첫 구현은 TypeScript 해석 함수가 맡는다.
- 결과는 파싱된 본문에 의존해야 한다. 본문이 바뀌면 후보와 근거도 바뀌어야 한다.
- 외부 LLM을 붙일 때도 같은 입력과 출력 기준을 유지한다.
- `EvidenceRef.snippet`은 실제 파싱된 본문 일부여야 한다.
- `src/server/trip-facts/extractTripFacts.ts`의 late arrival 고정값은 이 장면 기준과 맞지 않는다.
- Python baseline을 계속 쓴다면 같은 강릉 숙소 파일 기준으로 맞춘다.

## 이번 단계 밖

- 모든 숙소 필드 추출.
- conflict 전체 처리.
- 민감정보 표시 세부 정책.
- AI 답변 문장 스타일 고도화.

## 현재 코드에서 볼 곳

- `src/server/trip-facts/extractTripFacts.ts`: 현재 `22:00 이후 숙소 연락 필요`를 고정으로 만든다. 이 장면에서는 바꿔야 한다.
- `src/server/trip-facts/normalizeTripFacts.ts`: 근거가 없으면 `missing`으로 둘 수 있다.
- `src/ai/examples/accommodation_checkin.json`: 현재 late arrival 값이 있어 이번 자료 기준과 맞지 않는다.
- `src/ai/tests/test_baseline.py`: late arrival 값이 있다고 기대하는 테스트를 자료 기준으로 다시 봐야 한다.

## 확인 기준

1. 체크인 시작 시간은 `15:00`, `근거 있음`으로 나온다.
2. 체크인 근거는 강릉 숙소 파일에서 파싱한 본문 일부를 보여준다.
3. 늦은 도착 조건은 값 없이 `근거 부족`으로 나온다.
4. 파싱 본문에서 `15:00`을 제거하면 체크인 시간이 `근거 있음`으로 나오지 않는다.

## 남은 판단

- `근거 부족`을 후보로 직접 만들지, 질문 답변 단계에서 없는 항목으로 다룰지.
- TypeScript 구현과 Python baseline을 같은 작업에서 같이 맞출지.
