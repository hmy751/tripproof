# 숙소 체크인 03 - 자료함 채팅과 인라인 근거

상태: sub-spec draft.

부모: [숙소 체크인 확인 - Airbnb 강릉 숙소 파일](2026-06-06-accommodation-checkin-airbnb-gangneung.md)

## 왜 지금

사용자가 체감하는 첫 제품 동작은 채팅 답변이다. 이번 단계는 자료함에 질문했을 때 02의 해석 결과가 답변과 근거로 보이는 데까지를 닫는다.

## 사용자 장면

사용자가 전체 자료함에 묻는다.

```text
체크인은 몇 시부터야? 늦게 도착하면 어떻게 해야 해?
```

## 이번에 닫을 것

- 사용자 질문이 채팅에 남는다.
- 답변은 체크인 시간과 늦은 도착 조건을 함께 다룬다.
- 체크인 시작 시간은 `15:00`, `근거 있음`으로 보인다.
- 체크인 답변 안에서 근거 snippet을 볼 수 있다.
- 늦은 도착 조건은 값을 만들지 않고 `근거 부족`으로 보인다.
- 자료가 없거나 파싱 실패 상태이면 질문 흐름이 그 상태를 보여준다.

## 기술 기준

- `src/client/App.tsx`가 materials, parsed body, answer state를 연결한다.
- `ChatWorkspace`는 단순 문자열 답변만이 아니라 상태와 근거를 표시할 수 있어야 한다.
- 한 답변 안에서 항목별 상태를 나눠 보여준다.
- 답변의 근거 snippet은 02에서 받은 `EvidenceRef`를 사용한다.

## 이번 단계 밖

- 긴 대화 기억.
- 모든 질문 의도 분류.
- 답변 문장 스타일 고도화.
- 카드 초안 생성은 04에서 다룬다.

## 현재 코드에서 볼 곳

- `src/client/App.tsx`: 현재 자료가 없고 질문이 막힌다.
- `src/client/components/ChatWorkspace.tsx`: 현재 assistant 메시지는 단순 text 중심이다.
- `src/client/types.ts`: 근거와 상태가 붙은 답변을 표현할 수 있을지 확인이 필요하다.

## 확인 기준

1. 사용자가 질문하면 assistant 답변이 나온다.
2. 답변에 `15:00`과 `근거 있음`이 보인다.
3. 답변에 late arrival `근거 부족`이 보인다.
4. 체크인 근거 snippet을 답변 안에서 확인할 수 있다.

## 남은 판단

- 근거를 항상 펼쳐 보일지, 접었다 펼치는 UI로 둘지.
- 한 질문 안에서 두 항목을 한 답변 카드로 보여줄지, 항목별 작은 블록으로 나눌지.
