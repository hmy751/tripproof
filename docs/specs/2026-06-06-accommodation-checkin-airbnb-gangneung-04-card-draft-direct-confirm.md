# 숙소 체크인 04 - 카드 초안과 직접 확인

상태: sub-spec draft.

부모: [숙소 체크인 확인 - Airbnb 강릉 숙소 파일](2026-06-06-accommodation-checkin-airbnb-gangneung.md)

## 왜 지금

TripProof는 답변 로그만 쌓는 도구가 아니다. 사용자가 남길 정보를 카드 초안으로 올리고, AI가 못 찾은 값도 직접 확인으로 채울 수 있어야 한다. 이번 단계는 채팅 답변에서 카드 초안으로 넘어가는 흐름을 닫는다.

## 사용자 장면

사용자는 체크인 답변을 보고 `15:00`은 카드로 남기고 싶다. 늦은 도착 조건은 자료에 없지만, 호스트에게 확인한 뒤 직접 적어두고 싶다.

## 이번에 닫을 것

- 체크인 `15:00` 답변을 카드 초안으로 올릴 수 있다.
- 카드 초안에는 일정, 이름, 값, 상태가 보인다.
- 사용자가 초안의 이름이나 값을 고칠 수 있다.
- 늦은 도착 조건처럼 `근거 부족`인 항목도 직접 값을 입력할 수 있다.
- 직접 입력하거나 수정한 카드는 `직접 확인`으로 구분된다.
- 카드 초안은 대시보드에 자동 확정되지 않는다.

## 기술 기준

- `DraftListPanel`은 빈 화면이 아니라 초안 목록과 편집 상태를 가진다.
- draft state는 채팅 답변의 항목에서 만들어진다.
- `근거 있음` 초안과 `직접 확인` 초안은 출처 표시가 다르다.
- 직접 입력값은 AI 근거 snippet처럼 보이지 않게 한다.

## 이번 단계 밖

- 호스트에게 메시지를 보내는 기능.
- 모든 카드 편집 UX.
- 대시보드 배치는 05에서 다룬다.
- 민감정보 표시 세부 정책.

## 현재 코드에서 볼 곳

- `src/client/components/DraftListPanel.tsx`: 현재 항상 빈 초안이다.
- `src/client/App.tsx`: draft state가 아직 없다.
- `src/shared/tripFacts.ts`: 답변 상태와 카드 출처를 어떻게 나눌지 확인이 필요하다.

## 확인 기준

1. 체크인 답변에서 카드 초안을 만들 수 있다.
2. 초안에서 값과 이름을 확인하거나 수정할 수 있다.
3. `근거 부족` 항목도 직접 값을 채울 수 있다.
4. 직접 채운 카드는 `직접 확인`으로 표시된다.
5. 초안은 사용자가 확정하기 전까지 대시보드에 보이지 않는다.

## 남은 판단

- `근거 부족` 답변에서 바로 빈 초안을 만들지, 사용자가 먼저 `직접 확인으로 남기기`를 눌러야 할지.
- 카드 출처를 client state로만 둘지, shared 타입에 올릴지.
