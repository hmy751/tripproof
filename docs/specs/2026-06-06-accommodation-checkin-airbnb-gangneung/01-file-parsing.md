# 숙소 체크인 01 - 강릉 숙소 파일 파싱

상태: sub-spec draft.

부모: [숙소 체크인 확인 - Airbnb 강릉 숙소 파일](index.md)

## 왜 지금

제품의 시작은 사용자가 자료 파일을 넣고, 제품이 그 파일에서 본문을 얻는 것이다. 이 단계는 강릉 숙소 파일 두 개를 제품에 넣고, 파싱된 본문이 자료함과 AI 해석 입력으로 이어지는 데까지를 닫는다.

## 사용자 장면

사용자는 강릉 Airbnb 숙소의 `예약 확인서` 파일과 `호스트 메시지` 파일을 제품에 넣는다. 파일이 들어오면 사용자는 바로 체크인 질문을 할 수 있어야 한다.

```text
체크인은 몇 시부터야? 늦게 도착하면 어떻게 해야 해?
```

## 이번에 닫을 것

- 사용자가 강릉 숙소 파일 두 개를 선택하거나 추가할 수 있다.
- 제품은 각 파일의 본문을 읽는다.
- 자료함에는 파일명, 자료 종류, 파싱 상태가 보인다.
- 파싱된 본문은 AI 해석 단계가 읽을 수 있는 상태로 저장된다.
- `예약 확인서` 본문에는 체크인 시작 시간 `15:00`과 체크아웃 `11:00`이 들어간다.
- `호스트 메시지` 본문에는 late arrival 구체 조건을 넣지 않는다.
- 파일 파싱이 실패하면 실패 상태가 보이고, 조용히 성공한 것처럼 넘어가지 않는다.

## 기술 기준

- client는 React/Vite 화면에서 파일을 받는다.
- 첫 강릉 숙소 공개 파일은 `.txt` 또는 `.md`로 둔다.
- `.txt`/`.md` 파일은 브라우저 File API로 읽는다.
- 파일 읽기, 자료함 item 생성, AI 해석 입력 생성을 분리한다.
- 파일 본문 없이 질문에 답하지 않는다.

## 이번 단계 밖

- Gmail 계정에서 자동으로 가져오기.
- Airbnb 앱 화면 자동 수집.
- 모든 파일 형식 전체 지원.
- 파일 장기 저장과 계정 동기화.

## 확인 기준

1. 사용자가 강릉 숙소 파일 두 개를 넣을 수 있다.
2. 자료함에 두 파일이 파싱 완료 상태로 보인다.
3. 질문 입력이 자료 없음 상태로 막히지 않는다.
4. AI 해석 입력에는 실제 파싱된 본문이 들어간다.
5. 호스트 메시지에 `22:00 이후 연락`, `셀프 체크인 가능` 같은 late arrival 값이 없다.

## 남은 판단

- 첫 강릉 숙소 공개 파일을 `.txt`로 둘지, `.md`로 둘지.
- 파일명은 `gangneung-booking-confirmation` / `gangneung-host-message`처럼 둘지.
