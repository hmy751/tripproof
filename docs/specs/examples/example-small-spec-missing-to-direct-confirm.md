# Example: 작은 독립 spec - 근거 부족에서 직접 확인 카드로 올리기

이 문서는 작은 단위 작업도 spec이 될 수 있는 경우를 보여주는 calibration sample이다. 작은 수정마다 파일을 만들라는 뜻이 아니다. 이 예시는 상태 언어가 반복해서 drift될 때 독립 spec으로 남기는 정도를 보여준다.

## 왜 이 정도 spec을 남기는가

`근거 부족`, `확인 필요`, `직접 확인`은 서로 가까워 보여도 다른 축의 말이다. 이 구분이 흐려지면 AI가 못 찾은 정보를 사용자가 직접 채워 올리는 흐름이 막히거나, 반대로 사용자가 직접 채운 값이 AI 근거 카드처럼 보일 수 있다.

## Light Brief

```text
왜 지금: 근거 부족 답변과 직접 확인 카드의 상태 언어가 섞이면 사용자가 다음 행동을 이해하기 어렵다.
사용자 장면: 사용자가 "늦게 도착하면 어떻게 돼?"라고 묻지만 자료에 조건이 없고, 이후 직접 확인한 값을 카드로 올린다.
먼저 고를 slice: 이미 좁은 독립 경로라 구현면 펼치기는 생략하고, `근거 부족 답변 -> 직접 입력 -> 직접 확인 카드`만 화면까지 잇는다.
이번 AC: 근거 부족 답변은 값을 만들지 않음; 직접 입력한 카드는 직접 확인; 대시보드는 사용자 확정 뒤에만 표시
주의할 점: `확인 필요`와 `직접 확인`을 같은 말로 쓰지 않는다.
남은 판단: 카드 출처를 서버나 eval도 읽어야 하는 시점에 다시 판단한다.
```

## 왜 구현면 펼치기를 생략하는가

이 예시는 큰 feature가 아니라 상태 언어가 drift되는 작은 독립 spec이다. 이미 사용자 장면과 build path가 좁다.

```text
근거 부족 답변
-> 사용자가 직접 값 입력
-> 카드 초안 확정
-> 대시보드 출처가 직접 확인으로 표시
```

따라서 `자료 입력`, `AI 후보`, `정규화`, `채팅`, `카드`, `대시보드`를 다시 넓게 펼치지 않는다. 필요한 구현면은 위 경로 안에서만 확인한다. 만약 이 경로가 카드 편집 전체 UX, 서버나 eval에서 카드 출처를 읽는 흐름, 평가 기준까지 커지면 그때 별도 feature spec 후보로 본다.

## 제품 동작 기준

### 사용자 장면

사용자가 자료함 전체에 묻는다.

```text
호텔에 늦게 도착하면 어떻게 돼?
```

현재 자료에는 체크인 시작 시간만 있고, 늦은 도착 조건은 없다. 사용자는 이후 숙소에 직접 연락해서 `22:00 이후 도착 시 메시지 필요`라는 값을 확인한다.

### Goal

AI는 자료에 없는 늦은 도착 조건을 만들지 않고 `근거 부족`으로 멈춘다. 사용자는 직접 확인한 값을 카드 초안에 입력해 대시보드에 올릴 수 있고, 그 카드는 `직접 확인`으로 구분된다.

### Rules

- `근거 부족`은 답변 레벨의 EvidenceState다.
- `직접 확인`은 카드 레벨의 출처다.
- `근거 부족` 답변은 카드 올리기를 막지 않는다.
- 사용자가 직접 입력한 값에는 가짜 EvidenceRef를 붙이지 않는다.
- 사용자가 확정하지 않은 답변이나 초안은 대시보드에 나타나지 않는다.

### Non-goals

- 새로운 카드 편집 UI 전체를 설계하지 않는다. 단 직접 입력한 값이 카드 초안에서 대시보드 출처로 넘어가는 경계는 확인한다.
- 서버나 eval에서 카드 출처를 읽는 흐름은 이번 작업의 조건으로 삼지 않는다.
- 호스트에게 메시지를 보내는 기능은 직접 확인 카드가 화면에서 닫힌 뒤 붙인다.
- 예약번호, 출입 코드, 민감 정보 표시 기준은 별도 확인 흐름에서 다룬다.
- eval runner나 metric output을 만들지 않는다.

## 먼저 고를 slice

`근거 부족 답변 -> 직접 입력 -> 직접 확인 카드` 경로를 먼저 고른다. 이 slice는 자료에 없는 값을 AI가 만들지 않는 답변 레벨과, 사용자가 직접 확인한 값을 카드 출처로 구분하는 카드 레벨을 한 화면 흐름 안에서 분리한다.

이번에 직접 닫는 것은 상태 언어와 출처 경계다. 카드 편집 전체 UX, 호스트 연락 기능, 서버나 eval에서 카드 출처를 읽는 흐름은 화면에서 확인된 뒤 넓힌다.

## 이번 AC

1. 늦은 도착 조건이 자료에 없으면 답변은 `근거 부족`으로 표시되고 값은 비어 있다.
2. 사용자가 직접 값을 입력해 확정하면 대시보드 카드에는 `직접 확인`이 보인다.
3. 직접 확인 카드는 AI의 `근거 있음` 카드와 구분되며, 원문 근거가 있는 것처럼 보이지 않는다.

## 확인 방법

- 늦은 도착 조건이 없는 자료로 질문한다.
- 답변이 확정형 값이나 일반 지식으로 채워지지 않는지 본다.
- 사용자가 값을 입력해 카드 초안을 확정한 뒤 대시보드 문구가 `직접 확인`인지 본다.
- 값 입력 전에는 대시보드에 카드가 자동으로 생기지 않는지 본다.

## 실패 냄새

- `근거 부족` 답변을 사용자가 카드로 올릴 수 없게 막는다.
- 직접 입력한 카드가 `근거 있음`처럼 보인다.
- `확인 필요`를 "사용자가 직접 확인했다"는 뜻으로 쓴다.
- 서버나 eval에서 카드 출처를 읽는 설계가 먼저 커지고, 화면에서 사용자가 보는 동작이 뒤로 밀린다.
